# Documentación del Proyecto: Aurora Nova

Bienvenido a la documentación centralizada del proyecto Aurora Nova. Esta carpeta (`/docs`) sirve como la única fuente de verdad para todos los aspectos conceptuales, de diseño, arquitectura y funcionales del proyecto.

## Estructura de la Documentación

La información está organizada en las siguientes carpetas para facilitar el acceso y la consulta:

- **[01_requirements/](./01_requirements/)**: Contiene todos los requerimientos funcionales, no funcionales e historias de usuario que definen **qué** debe hacer el sistema.

- **[02_architecture/](./02_architecture/)**: Describe **cómo** está construido el sistema. Incluye decisiones de arquitectura (ADRs), diagramas de base de datos, flujos de datos y la justificación de las tecnologías utilizadas.

- **[03_features/](./03_features/)**: Detalla el funcionamiento de cada una de las funcionalidades principales de la aplicación. Cada documento aquí es una especificación de una _feature_ concreta.

## Cómo Contribuir

- Para nuevos requerimientos, añadir un documento en `01_requirements/`.
- Para decisiones de arquitectura importantes, crear un nuevo ADR en `02_architecture/`.
- Para definir una nueva funcionalidad, crear su archivo correspondiente en `03_features/`.
- Siempre que se cree una nueva carpeta, añadir un `README.md` que explique su propósito.

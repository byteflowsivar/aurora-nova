# 01. Requerimientos

Esta carpeta contiene todos los documentos relacionados con los requerimientos del proyecto.

## Contenido

Aquí se deben almacenar:

- **Requerimientos Funcionales:** Documentos que describen el comportamiento del sistema, es decir, qué es lo que el sistema debe hacer. Por ejemplo: `RF-001_Gestion_de_Usuarios.md`.

- **Requerimientos No Funcionales:** Documentos que describen las características de calidad del sistema, como rendimiento, seguridad, escalabilidad, etc. Por ejemplo: `RNF-001_Seguridad_de_Contraseñas.md`.

- **Historias de Usuario:** Si se utiliza una metodología ágil, aquí se pueden documentar las historias de usuario. Por ejemplo: `HU-001_Crear_Usuario.md`.

## Propósito

El objetivo de esta carpeta es centralizar la definición de **qué** se debe construir, sirviendo como referencia para el desarrollo y las pruebas.

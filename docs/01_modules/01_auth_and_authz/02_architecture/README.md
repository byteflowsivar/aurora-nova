# 02. Arquitectura

Esta carpeta documenta la arquitectura del sistema y las decisiones técnicas clave que la definen.

## Contenido

Aquí se deben almacenar:

- **Decisiones de Arquitectura (ADRs - Architecture Decision Records):** Documentos cortos y específicos que capturan una decisión arquitectónica importante, su contexto y sus consecuencias. Este es el lugar para justificar por qué se eligió una tecnología o un patrón de diseño sobre otro. Por ejemplo: `ADR-001_Eleccion_de_NextJS_como_framework.md`.

- **Diagramas de Sistema:** Diagramas de alto nivel que muestran la estructura general del sistema, sus componentes y cómo interactúan. Por ejemplo: `Diagrama_General_Componentes.png`.

- **Diagramas de Base de Datos:** El esquema de la base de datos (diagrama Entidad-Relación) y cualquier otra documentación relacionada con la persistencia de datos.

- **Flujos de Datos:** Diagramas que ilustran cómo fluye la información a través del sistema para procesos clave.

## Propósito

El objetivo de esta carpeta es explicar **cómo** está construido el sistema y **por qué** se construyó de esa manera. Sirve como una guía fundamental para los desarrolladores actuales y futuros.
